# ML/AI pipeline

## Description

A system that uses machine learning or artificial intelligence to process data, make predictions, or generate content.

## Typical data

- training datasets (may contain personal data)
- model inputs and outputs
- user interaction data
- behavioral data for model improvement

## Common regulatory triggers

- personal data in training data (152-ФЗ)
- automated decision-making considerations
- cross-border transfer of training data
- data subject rights (explanation, deletion)

## Common red flags

- personal data in training sets without consent
- no data anonymization before training
- model outputs that reveal personal data
- no human review of automated decisions
- training data stored in foreign cloud

## Recommended controls

- anonymize or pseudonymize training data
- document data sources and consent
- implement model output filtering
- provide human review for significant decisions
- store training data in compliant location
- regular bias and fairness audits
